The Everflow API is organized around REST. It uses standard HTTP verbs, returns JSON responses, and uses conventional HTTP status codes.

## Base URL

All API requests should be made to:

```
https://api.eflow.team/v1
```

A small subset of EU-hosted accounts use `https://api-eu.eflow.team/v1` instead.

## Authentication

Authenticate every request by including your API key in the `X-Eflow-API-Key` header.

```bash
curl -H "X-Eflow-API-Key: <your-api-key>" \
  https://api.eflow.team/v1/networks/offers
```

API keys are generated in the Everflow UI under **Control Center > Security**. See [Authentication](/user-guide/authentication) for details.

## Available APIs

| API | Description |
|-----|-------------|
| **Network API** | Full platform management: offers, affiliates, advertisers, reporting, tracking, and more. |
| **Affiliate API** | Affiliate-scoped access to reporting, deals, invoices, and traffic management. |
| **Advertiser API** | Advertiser-scoped access to offer and reporting management. |
| **Marketplace API** | Partner connections, earnings, and marketplace functionality. |
| **Metadata API** | Reference data: countries, currencies, devices, timezones, and more. |

This documentation currently covers the **Network API**, which provides the most comprehensive access to the Everflow platform.
