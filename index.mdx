The Everflow API is organized around REST. It uses standard HTTP verbs, returns JSON responses, and uses conventional HTTP status codes.

## Base URL

All API requests should be made to:

```
https://api.eflow.team/v1
```

A small subset of EU-hosted accounts use `https://api-eu.eflow.team/v1` instead.

## Authentication

Authenticate every request by including your API key in the `X-Eflow-API-Key` header.

```bash
curl -H "X-Eflow-API-Key: <your-api-key>" \
  https://api.eflow.team/v1/networks/offers
```

API keys are generated in the Everflow UI under **Control Center > Security**. See [Authentication](/user-guide/authentication) for details.

## Available APIs

| API | Description |
|-----|-------------|
| **[Network API](/api-reference/post-networksofferstable)** | Full platform management: offers, affiliates, advertisers, reporting, tracking, and more. |
| **[Affiliate API](/api-reference/get-affiliatesoffersrunnable)** | Affiliate-scoped access to offers, reporting, deals, invoices, postbacks, and traffic management. |
| **[Advertiser API](/api-reference/get-advertisersoffers)** | Advertiser-scoped access to offers and reporting. |
| **[Marketplace API](/api-reference/post-partnersconnectionstable)** | Partner connections, offers, earnings, and API key management. |
| **[Webhooks](/webhooks/overview)** | Receive real-time HTTP notifications when advertisers, offers, or partners change. |
| **[JavaScript SDK](/sdk/overview)** | Client-side click, conversion, and impression tracking without redirect links. |
